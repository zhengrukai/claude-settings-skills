## 回答结构规范

每个知识点的完整面试答案应包含（根据深度要求裁剪）：

```
1. 一句话定义（先给结论）
2. 核心原理/底层实现（是什么、怎么做的）
3. 关键特性/设计亮点（为什么这样设计）
4. 与相似概念的对比（和 X 的区别）
5. 使用场景与注意事项（什么时候用、踩坑点）
6. 高频追问及参考回答（面试官下一步会问什么）
```

## 质量标准

### ✅ 好答案的特征
- **结论前置**：第一句话给出核心结论，不绕弯子
- **有数字/数据**：如 HashMap 默认容量 16、负载因子 0.75、树化阈值 8
- **讲原理不背概念**：说明"为什么"而不只是"是什么"
- **有对比维度**：和相近技术做横向对比
- **提供记忆锚点**：帮助面试者记住核心点的简洁总结

### ❌ 避免的问题
- 只给定义不讲原理
- 遗漏高频追问点（面试官必问的点不能漏）
- 没有对比分析（比如只讲 synchronized 不提 ReentrantLock）
- 答案太长没有重点（面试场景下需要层次分明）

## 示例参考

**输入**：HashMap 的 put 流程帮我补充一下

**输出结构**：
```
### 一句话结论
HashMap.put() 的本质是：计算 key 的 hash → 定位数组桶 → 处理冲突（链表/红黑树）→ 超阈值扩容。

### 详细流程（Java 8）
1. 调用 key.hashCode()，再做高位扰动：(h = key.hashCode()) ^ (h >>> 16)
2. 通过 (n-1) & hash 定位数组下标（n 为数组长度，必须是 2 的幂）
3. 桶为空 → 直接插入；不为空 → 遍历链表/红黑树
4. 链表长度 ≥ 8 且数组长度 ≥ 64 → 转为红黑树
5. size > capacity × loadFactor(0.75) → 触发 resize()，容量翻倍

### 设计亮点
- 高位扰动：减少低位相同时的 hash 冲突
- 2 的幂次容量：用位运算代替取模，性能更好
- 树化阈值 8：泊松分布下链表超过 8 的概率极低（< 千万分之一）

### 高频追问
1. 为什么容量必须是 2 的幂？
2. 扰动函数为什么只异或高 16 位？
3. 树退化回链表的阈值是多少？（6）
4. JDK 7 和 JDK 8 的 put 有什么区别？（头插法 vs 尾插法）
5. 并发场景下 put 会有什么问题？（引出 ConcurrentHashMap）
```
